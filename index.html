Website Learning Material

這幾天的工作就是先學習。使用網站為MDN Web Docs中的Learn web development。下面列了些我覺得必須要看的一些文章。裡面的資料非非非非非常詳細，甚至有非常多的超連結，可以先不要要點那些，因為會看到心很累，資訊量太大了。把下面這些文章看完之後，應該會建立了一個非常不錯的開發環境，並知道如何發布(部署)。

Install VSCode

雖然天下編輯程式碼的工具百百種，但設計網頁真的會需要運用到許多不同語言，使用 VSCode 真的會有一種很爽的感覺。還是下載一下吧。
download VSCode

Command Line：

在做網頁時，需要測試一些東西，或是引用一些package(別人使用，我們引用的)，這些都需要用到cmd(windows+r，輸入cmd可開啟)，操控cmd的指令就叫做command line

cmd 的速成小教學

Package Management：

寫網頁的時候可以比較容易管理使用的package，檢查更新或是路徑重複等。同時也可以自定命令項及及時自動更新本地伺服器的網頁(不需要一直切來切去按F5)

npm 教學 (一種常見的package manager)

配置初始項目

前面的東西都看過，並實際操作過，就可以開始配置項目了。要記得的是前面文章所配置的東西都需要重複配置，不然套件會無法使用，例如一定要記得重新 npm install -g parcel-bundler
從Configuring the initial project開始看

假如要 push 到 Github 時，遇到 github push failed (remote: Permission to userA/repo.git denied to userB.)

這種情況通常是因為這台電腦可能同時使用超過一支 Github account，所以需要梗改一下設定，記得要再改回來就是。
可以參考這篇文章的作法

假如說需要測試的部分比較多可以試試…(這篇可以選讀)

這篇文章主要是在闡述如何可以快速的 deploy 網站
