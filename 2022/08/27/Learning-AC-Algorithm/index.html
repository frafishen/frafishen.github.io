Learning AC Algorithm

在看NLP入門書的時候，真的發現有很多演算法以及資料結構需要補強。在看字串匹配的時候，看到AC自動機，頭直接差點快要炸掉，原來是因為有很多東西，我直接跳著看，所以這編列一些我的學習歷程。

處理字串匹配問題之前，除了最暴力直接枚舉匹配，複雜度直上O(m * n)，先學學資料結構

Trie 字典樹

以字典數對字串做儲存的話，可以減少儲存空間，並些讓 travel 及 insert 的時間複雜度降到線性

程式碼

在學 AC 自動機之前，還有個演算法要先看

KMP

Hashing Method 用來 access 字串不是不行，但它畢竟是一個隨機演算法，且模運算其實很花時間。
KMP = Knuth-Morris-Pratt algorithm
